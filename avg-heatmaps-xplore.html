1 Goal

The goal of this exercise is to create a time series of heat maps, as automated as possible.

The metric being visualized is total number of new observations in a time window of 3 days.

To create a time series of heat maps, we need a complete time series of trap counts for all locations.


Challenges with our data:

  • date range varies for the 5 traps involved

  • missing dates exist for all traps

Define the dates when the sticky papers were replaced (i.e.g, the cummulative count gets reset). These come from a spreadsheet by C. Burke. Note also that all of the traps we are working with were reset on these dates.

reset_count_dt <- as.Date(c("2023-08-09",
                            "2023-08-15",
                            "2023-08-24",
                            "2023-08-29",
                            "2023-09-06"))

Because our goal is to create a time series of heat maps, we need to define a date range where all (or nearly all) of the traps have data. The table above suggests we should start on 8/15/2023.

As for the end date, we know the traps were reset on 9/6/2023. There are no maintenance records after that, however by manually checking the data on the data dashboard, we can see that several of the traps have very spotty data after September 5 (due to technical issues and harvest). Therefore, we will use an end date of September 5, 2023:

tc3_tbl <- tc2_tbl |>

5 Compute number of new observations

For each trap, we need to identify the first date after the sticky papers were changed:

## Add a column for the amount to subtract for the daily count
tc5_tbl <- tc4_tbl |> 
  arrange(device_id, date) |> 
  mutate(subtract_out = lag(count, 1))

## Set the 'subtract_out' to zero, the day after the paper is changed
tc6_tbl <- tc5_tbl |> 
  mutate(subtract_out = if_else(date %in% (reset_count_dt + 1),
                                0,
                                subtract_out))


Compute a 3-day rolling sum:

tc7_tbl <- tc6_tbl |> 
  mutate(new_obs = count - subtract_out)
